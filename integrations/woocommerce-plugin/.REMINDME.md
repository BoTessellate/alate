# WooCommerce Plugin Setup Instructions

## Prerequisites

1. **WordPress Installation**
   - WordPress 5.8 or higher
   - PHP 7.4 or higher

2. **WooCommerce**
   - WooCommerce 5.0 or higher installed and active

## Installation

### Method 1: Upload via WordPress Admin

1. Download the `woocommerce-plugin` folder
2. Rename to `moodlayer`
3. Zip the folder: `moodlayer.zip`
4. Go to WordPress Admin → Plugins → Add New → Upload Plugin
5. Select `moodlayer.zip` and install
6. Activate the plugin

### Method 2: FTP/SFTP Upload

1. Connect to your server via FTP/SFTP
2. Navigate to `/wp-content/plugins/`
3. Upload the `woocommerce-plugin` folder
4. Rename to `moodlayer`
5. Go to WordPress Admin → Plugins
6. Find "Mood Layer" and click Activate

### Method 3: Composer (Advanced)

```bash
# In your WordPress project
composer require moodlayer/woocommerce-plugin
```

## Configuration

### 1. Get API Credentials

1. Log in to your Mood Layer dashboard
2. Go to Settings → API Keys
3. Generate or copy your API Key and Secret

### 2. Configure Plugin

1. Go to WordPress Admin → Settings → Mood Layer
2. Enter your API Key
3. Enter your API Secret
4. Enter your Brand ID (from Mood Layer dashboard)
5. Choose Sync Mode:
   - **Manual**: Sync only when triggered manually
   - **Automatic**: Sync on every product change
6. Click "Save Changes"

### 3. Test Connection

1. On the Mood Layer settings page
2. Click "Test Connection"
3. Should show "Connection successful!"

## Environment Variables (Alternative)

Instead of configuring via admin, you can use `wp-config.php`:

```php
// Mood Layer Configuration
define('MOODLAYER_API_KEY', 'your_api_key_here');
define('MOODLAYER_API_SECRET', 'your_api_secret_here');
define('MOODLAYER_BRAND_ID', 'your_brand_id_here');
```

Then in your theme's `functions.php`:

```php
add_filter('pre_option_moodlayer_api_key', function() {
    return defined('MOODLAYER_API_KEY') ? MOODLAYER_API_KEY : false;
});
add_filter('pre_option_moodlayer_api_secret', function() {
    return defined('MOODLAYER_API_SECRET') ? MOODLAYER_API_SECRET : false;
});
add_filter('pre_option_moodlayer_brand_id', function() {
    return defined('MOODLAYER_BRAND_ID') ? MOODLAYER_BRAND_ID : false;
});
```

## API Endpoints

The plugin registers these REST API endpoints:

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/wp-json/moodlayer/v1/ping` | GET | Health check |
| `/wp-json/moodlayer/v1/health` | GET | Detailed health check |
| `/wp-json/moodlayer/v1/sync` | POST | Trigger product sync |
| `/wp-json/moodlayer/v1/products` | GET | Get all products |

### Authentication

All endpoints (except `/health`) require authentication:

```
Authorization: Bearer {api_key}:{hmac_signature}
```

The signature is computed as:
```
HMAC-SHA256(request_body, api_secret)
```

## Testing

### Test Ping Endpoint

```bash
# Public health check (no auth required)
curl https://your-store.com/wp-json/moodlayer/v1/health

# Authenticated ping
API_KEY="your_key"
API_SECRET="your_secret"
SIGNATURE=$(echo -n "" | openssl dgst -sha256 -hmac "$API_SECRET" | cut -d' ' -f2)
curl -H "Authorization: Bearer $API_KEY:$SIGNATURE" \
  https://your-store.com/wp-json/moodlayer/v1/ping
```

### Test Sync Endpoint

```bash
API_KEY="your_key"
API_SECRET="your_secret"
BODY='{"limit":10}'
SIGNATURE=$(echo -n "$BODY" | openssl dgst -sha256 -hmac "$API_SECRET" | cut -d' ' -f2)
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $API_KEY:$SIGNATURE" \
  -d "$BODY" \
  https://your-store.com/wp-json/moodlayer/v1/sync
```

## Troubleshooting

### "WooCommerce Required" Error

- Ensure WooCommerce is installed and activated
- Check WooCommerce version is 5.0 or higher

### "API credentials not configured" Warning

- Go to Settings → Mood Layer
- Enter your API Key and Secret
- Click Save Changes

### "Connection failed" on Test

- Verify API credentials are correct
- Check your server can make outbound HTTPS requests
- Ensure Mood Layer API is accessible from your server

### Products Not Syncing

- Check Sync Mode is set to "Automatic"
- Or manually click "Sync All Products"
- Check error logs in WordPress debug.log

### REST API 404 Errors

- Ensure permalinks are set (Settings → Permalinks → Save)
- Check .htaccess file is writable
- Verify REST API is not disabled
